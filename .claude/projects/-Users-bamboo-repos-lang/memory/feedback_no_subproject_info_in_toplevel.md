Don't put sub-project specific information into top-level files (including CLAUDE.md). Instead reference subfolders for more info. Put project-specific details into corresponding README.md files in subdirs.

**Why:** User prefers clean separation — top-level files should only provide an index/overview, not duplicate what belongs in each subproject.

**How to apply:** When creating or editing top-level docs, keep them as brief pointers to subdirectories. If detailed info is needed for a subproject, add it to that subproject's own README.md or CLAUDE.md.
